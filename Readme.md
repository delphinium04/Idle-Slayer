# Idle Slayer — 방치형 RPG 포트폴리오

## 기획

### 핵심 컨셉

모바일 방치형 RPG
자동 전투 + 성장 중심 구조
캐릭터는 다양 / 스탯은 공통 / 전투 방식만 차이

### 기본 게임 루프

전투 → 골드 획득 → 스탯 강화 → 더 강한 적 → 반복

### 캐릭터 시스템

여러 캐릭터 선택 가능
차이점: 기본 공격 방식, 스킬
공통: 동일한 스탯 구조

### 스탯 구조

모든 스탯은 골드로 강화
- 공격력
- 크리티컬 확률
- 크리티컬 데미지
- 공격 속도
- 체력

## 핵심 시스템

- 자동 전투 (공격 속도 기반)
- 데미지 계산 (크리티컬 포함)
- 골드 획득
- 스탯 성장
- 오프라인 보상 ⭐
- 캐릭터 다양화
- 스킬 시스템
- UI 개선
- 데이터 기반 구조

### 데미지 계산식

- 데미지: AttackPower
- 크리티컬 확률: Random(0~1):Float < CriticalChance
- 최종 데미지: IsCritical ? Damage × (1 + CriticalDamage / 100) : Damage
- 기본 공격 속도 계산식: 1 / AttackSpeed

## 개발 단계

### 1단계: MVP (가장 중요)

목표: “돌아가는 게임 만들기”

- 캐릭터 1명
- 적 1종
- 자동 전투
- 골드 획득
- 공격력 강화 1개
- 최소 UI

목표: 게임 완성

### 2단계: 핵심 시스템

“포트폴리오 수준까지 만들기”

- 크리티컬 시스템
- 공격 속도
- 체력 시스템
- 저장/불러오기
- 오프라인 보상 ⭐
- UI 개선
- DPS 표시

### 3단계: 확장 및 차별화

- 캐릭터 추가 (2~3개)
- 스킬 시스템
- 데이터 온라인 동기화
- 이벤트 기반 구조
- 최적화 및 리팩토링 (Update 최소화)
- 모바일 플랫폼을 고려한 최적화

## Git prefix 키워드

- feat:     기능 추가
- fix:      버그 수정
- refactor: 구조 개선 (동작 변화 없음)
- test:     테스트 코드
- chore:    기타 작업 (설정, 빌드 등)
- docs:     문서 수정
- style:    코드 스타일 (포맷팅 등)
- D:        단계적 계획의 마무리 Prefix
